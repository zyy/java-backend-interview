# SOLID 原则

> 面向对象设计原则

## 🎯 面试重点

- 五大原则含义
- 实际应用

## 📖 五大原则

### 单一职责 (SRP)

```java
/**
 * 单一职责
 */
public class SRP {
    // 一个类只负责一个职责
    // 修改原因只有一个
}
```

### 开闭原则 (OCP)

```java
/**
 * 开闭原则
 */
public class OCP {
    // 对扩展开放，对修改关闭
    // 使用抽象和多态
}
```

### 里氏替换 (LSP)

```java
/**
 * 里氏替换
 */
public class LSP {
    // 子类可以替换父类
    // 不改变程序正确性
}
```

### 接口隔离 (ISP)

```java
/**
 * 接口隔离
 */
public class ISP {
    // 不应该依赖不使用的接口
    // 接口要小而专
}
```

### 依赖倒置 (DIP)

```java
/**
 * 依赖倒置
 */
public class DIP {
    // 依赖抽象，不依赖具体
    // 面向接口编程
}
```

### Q1: 举例说明开闭原则？

**答：** 策略模式新增策略只需实现接口，不修改已有代码。
