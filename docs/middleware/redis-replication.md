# Redis 主从复制与哨兵

> Redis 高可用方案

## 🎯 面试重点

- 主从复制原理
- 哨兵机制
- 故障转移

## 📖 主从复制

### 复制原理

```java
/**
 * 主从复制
 * 
 * 流程：
 * 1. 从节点发送 PSYNC 命令
 * 2. 主节点 BGSAVE 生成 RDB
 * 3. 发送 RDB 到从节点
 * 4. 主节点发送增量数据
 */
```

### 配置

```java
/**
 * 配置
 */
public class ReplicationConfig {
    // 主节点配置
    /*
     * replicaof 127.0.0.1 6379
     * 
     * 或
     * replicaof masterip masterport
     */
}
```

## 📖 哨兵机制

### 哨兵功能

```java
/**
 * 哨兵功能
 * 
 * 1. 监控：检查主从节点是否正常
 * 2. 通知：故障时通知
 * 3. 自动故障转移
 *   - 选择从节点为新主节点
 *   - 通知其他从节点
 *   - 客户端重新路由
 */
```

## 📖 面试真题

### Q1: 主从复制丢失数据？

**答：** 
- 异步复制，主节点故障可能丢失数据
- 可以配置 min-slaves-write-timeout
