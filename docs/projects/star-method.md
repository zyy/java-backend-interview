# STAR 法则

> 项目介绍方法论

## 📖 STAR 法则

```
S (Situation) - 情境
  描述项目背景、业务场景

T (Task) - 任务
  你负责什么、目标是什么

A (Action) - 行动
  你做了什么、怎么做的

R (Result) - 结果
  取得什么成果、量化指标
```

## 📖 示例

```
S: 电商系统日均订单 10 万单，大促时订单量激增

T: 负责订单系统性能优化，目标是将接口响应时间从 500ms 降到 100ms

A: 
  1. 分析慢查询，添加索引
  2. 引入 Redis 缓存热点数据
  3. 异步处理非核心逻辑
  4. 分库分表优化

R: 
  - 接口响应时间降至 80ms
  - 系统承载能力提升 5 倍
  - 大促期间零故障
```
