# 设计一个短链系统

> 系统设计题

## 🎯 需求

- 将长 URL 转换为短 URL
- 访问短 URL 跳转到长 URL
- 高并发、高可用

## 📖 设计方案

### 短链生成

```
方法1：自增ID + Base62编码
- 优点：简单
- 缺点：暴露业务量

方法2：Hash + 冲突处理
- MurmurHash + Base62
- 冲突时追加随机数重试

方法3：分布式ID
- 雪花算法生成 ID
- Base62 编码
```

### 存储方案

```
- Redis 缓存热点数据
- MySQL 持久化
- 分库分表应对数据量增长
```

### 架构

```
用户 -> CDN -> Nginx -> 网关 -> 短链服务 -> Redis -> DB
```
